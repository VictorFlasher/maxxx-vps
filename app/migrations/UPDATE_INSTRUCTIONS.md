# Инструкция по обновлению базы данных

## Шаг 1: Выполните миграцию для добавления колонки `original_filename`

Эта миграция добавит новую колонку для хранения оригинальных имён файлов.

```bash
psql -U postgres -d postgres -f app/migrations/add_original_filename_to_messages.sql
```

Или выполните SQL вручную через pgAdmin/другой клиент:

```sql
-- Добавляем колонку original_filename для хранения оригинального имени файла
ALTER TABLE maxxx_local.messages 
ADD COLUMN IF NOT EXISTS original_filename VARCHAR(255);

-- Индекс для поиска по оригинальному имени (опционально)
CREATE INDEX IF NOT EXISTS idx_messages_original_filename 
ON maxxx_local.messages(original_filename) 
WHERE original_filename IS NOT NULL;

-- Комментарий к колонке
COMMENT ON COLUMN maxxx_local.messages.original_filename IS 'Оригинальное имя файла для отображения и скачивания';
```

## Шаг 2: Перезапустите сервер

После применения миграции перезапустите ваш FastAPI сервер:

```bash
# Остановите текущий процесс (Ctrl+C)
# Затем запустите снова
uvicorn app.main:app --reload
```

## Что изменилось

### В базе данных:
- ✅ Добавлена колонка `original_filename VARCHAR(255)` в таблицу `messages`
- ✅ Создан индекс для быстрого поиска по оригинальным именам файлов
- ✅ Поле `content` теперь может быть NULL (для сообщений с файлами)

### В коде:
- ✅ При загрузке файла сохраняется его оригинальное имя
- ✅ Файлы на диске хранятся с зашифрованными именами (UUID)
- ✅ При отображении показывается оригинальное имя файла
- ✅ При скачивании файл получает своё оригинальное имя

### Для пользователя:
- 📎 Вместо "📎 Скачать файл" отображается имя файла, например `readme.txt`
- 📥 При клике файл скачивается с правильным именем
- 🔒 На диске файлы всё ещё защищены (имена UUID)

## Проверка работы

1. Откройте чат
2. Загрузите файл с понятным именем (например, `документ.pdf`)
3. Убедитесь, что в чате отображается имя файла
4. Кликните на файл — он должен скачаться с оригинальным именем

## Обратная совместимость

Код автоматически адаптируется под разные версии структуры БД:
- Если колонки `original_filename` нет → используется старый формат
- Если колонки `file_path`/`file_type` нет → используется content
- Старые сообщения корректно отображаются благодаря конвертации "на лету"
