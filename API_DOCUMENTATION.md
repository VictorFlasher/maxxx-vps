## Общие сведения

**Базовый URL:** `http://127.0.0.1:8000`

**Аутентификация:** Все API-запросы (кроме регистрации и входа) требуют JWT-токен в заголовке:
```
Authorization: Bearer <ваш_токен>
```

**Получение токена:** После успешного входа через `/api/login` сервер возвращает токен, который нужно сохранять и использовать для всех последующих запросов.

---

## Публичные маршруты (Frontend)

Эти маршруты открывают HTML-страницы в браузере.

| Метод | Путь | Описание | Доступ |
|-------|------|----------|--------|
| GET | `/` | Страница входа (login.html) | Все |
| GET | `/register` | Страница регистрации | Все |
| GET | `/chat` | Основная страница чата | Авторизованные |
| GET | `/chat?chat_id={id}` | Открытие конкретного чата | Авторизованные |
| GET | `/search-users` | Поиск пользователей | Авторизованные |
| GET | `/admin` | **Админ-панель** | Только админы |

## API Аутентификации

### Регистрация нового пользователя
**POST** `/api/register`

**Тело запроса:**
```json
{
  "username": "string",
  "email": "user@example.com",
  "password": "string"
}
```

**Ответ:**
- `200`: `{"message": "Пользователь создан"}`
- `400`: Ошибка валидации (email занят, неверный формат)
- `429`: Превышен лимит запросов (5 в минуту)

---

### Вход в систему
**POST** `/api/login`

**Тело запроса:**
```json
{
  "email": "user@example.com",
  "password": "string"
}
```

**Ответ:**
- `200`: `{"access_token": "eyJ...", "token_type": "bearer"}`
- `401`: Неверный email или пароль
- `429`: Превышен лимит запросов (10 в минуту)

**Важно:** Сохраните `access_token` для всех последующих запросов.

## API Чата

### Получить список чатов пользователя
**GET** `/api/chats`

**Ответ:** Список чатов с информацией о последних сообщениях.

---

### Отправить сообщение
**POST** `/api/messages`

**Тело запроса:**
```json
{
  "chat_id": 1,
  "text": "Привет!",
  "file_path": null,
  "file_type": null
}
```

---

### Редактировать сообщение
**PUT** `/api/messages/{message_id}`

**Тело запроса:**
```json
{
  "text": "Новый текст сообщения"
}
```

**Особенности:**
- Можно редактировать только свои сообщения
- Сообщение помечается флагом `is_edited`
- Изменения мгновенно рассылаются всем участникам чата через WebSocket

---

### Пожаловаться на сообщение
**POST** `/api/reports`

**Тело запроса:**
```json
{
  "message_id": 123,
  "reason": "Оскорбления"
}
```

**Ответ:** Жалоба попадает в админ-панель на рассмотрение.

---

### Создать групповой чат
**POST** `/api/chats/create-group`

**Тело запроса:**
```json
{
  "name": "Название группы",
  "member_ids": [2, 3, 4]
}
```

---

### Покинуть/удалить чат
**POST** `/api/chats/{chat_id}/leave`

**Логика:**
- Если вы последний участник группового чата → чат удаляется
- Если есть другие участники → вы покидаете чат
- Для личных чатов → чат скрывается у текущего пользователя

---

### Переименовать групповой чат
**PUT** `/api/chats/{chat_id}/rename`

**Тело запроса:**
```json
{
  "name": "Новое название"
}
```

**Доступ:** Только создатель/владелец чата.

## API Администрирования

**Внимание:** Все эндпоинты этой секции доступны **только пользователям с флагом `is_admin=true`**.

### Админ-панель (HTML)
**GET** `/admin`

Открывает веб-интерфейс администратора с вкладками:
- 📋 Жалобы
- 🚫 Активные баны
- 📜 История банов

---

### Забанить пользователя
**POST** `/api/admin/ban`

**Тело запроса:**
```json
{
  "user_id": 5,
  "reason": "Нарушение правил"
}
```

**Проверки:**
- Нельзя забанить себя
- Нельзя забанить другого админа
- Причина бана обязательна

**Ответ:**
```json
{
  "status": "success",
  "banned_user_id": 5,
  "reason": "Нарушение правил"
}
```

---

### Разбанить пользователя
**POST** `/api/admin/unban`

**Тело запроса:**
```json
{
  "user_id": 5
}
```

**Ответ:**
```json
{
  "status": "success",
  "unbanned_user_id": 5
}
```

---

### Получить активные баны
**GET** `/api/admin/bans`

**Ответ:**
```json
[
  {
    "ban_id": 1,
    "user_id": 5,
    "username": "baduser",
    "banned_by": 1,
    "banned_by_username": "admin",
    "reason": "Нарушение правил",
    "created_at": "2025-01-15T10:30:00Z"
  }
]
```

---

### Получить историю банов
**GET** `/api/admin/ban-history`

**Параметры query:**
- `user_id` (optional): фильтр по конкретному пользователю
- `limit` (default: 50): максимум записей

**Примеры:**
- `/api/admin/ban-history` — вся история
- `/api/admin/ban-history?user_id=5` — история пользователя 5
- `/api/admin/ban-history?user_id=5&limit=10` — последние 10 записей

**Ответ:**
```json
[
  {
    "history_id": 1,
    "user_id": 5,
    "username": "baduser",
    "action": "ban",
    "performed_by": 1,
    "performed_by_username": "admin",
    "reason": "Нарушение правил",
    "created_at": "2025-01-15T10:30:00Z"
  },
  {
    "history_id": 2,
    "user_id": 5,
    "username": "baduser",
    "action": "unban",
    "performed_by": 1,
    "performed_by_username": "admin",
    "reason": null,
    "created_at": "2025-01-16T12:00:00Z"
  }
]
```

---

### Получить жалобы
**GET** `/api/admin/reports`

**Параметры query:**
- `status` (default: 'pending'): `'pending'`, `'reviewed'`, `'dismissed'`, `'actioned'`
- `limit` (default: 50)

**Пример:** `/api/admin/reports?status=pending&limit=20`

**Ответ:**
```json
[
  {
    "report_id": 1,
    "message_id": 123,
    "reporter_id": 10,
    "reason": "Оскорбления",
    "status": "pending",
    "created_at": "2025-01-15T14:20:00Z",
    "message_text": "Текст сообщения...",
    "sender_id": 5,
    "chat_id": 2
  }
]
```

---

### Обработать жалобу
**POST** `/api/admin/reports/review`

**Тело запроса (отклонить жалобу):**
```json
{
  "report_id": 1,
  "action": "dismiss"
}
```

**Тело запроса (забанить автора):**
```json
{
  "report_id": 1,
  "action": "ban",
  "ban_reason": "Нарушение правил сообщества"
}
```

**Ответ:**
```json
{
  "status": "success",
  "action": "ban"
}
```

## База данных

### Схема БД
Все таблицы находятся в схеме `maxxx`.

### Основные таблицы

#### `users` — Пользователи
| Колонка | Тип | Описание |
|---------|-----|----------|
| user_id | SERIAL | ID пользователя |
| username | VARCHAR(50) | Уникальное имя |
| email | VARCHAR(255) | Уникальный email |
| password_hash | VARCHAR(255) | Хеш пароля |
| is_admin | BOOLEAN | Флаг администратора |
| is_banned | BOOLEAN | Текущий статус бана |
| created_at | TIMESTAMP | Дата регистрации |
| last_seen | TIMESTAMP | Последний вход |

---

#### `bans` — Активные баны
| Колонка | Тип | Описание |
|---------|-----|----------|
| ban_id | SERIAL | ID бана |
| user_id | INTEGER | Забаненный пользователь |
| banned_by | INTEGER | Кто забанил (админ) |
| reason | TEXT | Причина бана |
| created_at | TIMESTAMP | Дата бана |

---

#### `ban_history` — История банов (аудит)
| Колонка | Тип | Описание |
|---------|-----|----------|
| history_id | SERIAL | ID записи |
| user_id | INTEGER | Пользователь |
| action | VARCHAR(20) | `'ban'` или `'unban'` |
| performed_by | INTEGER | Кто выполнил действие |
| reason | TEXT | Причина (для бана) |
| created_at | TIMESTAMP | Дата действия |

**Важно:** Эта таблица хранит полную историю, даже после разбана.

---

#### `chats` — Чаты
| Колонка | Тип | Описание |
|---------|-----|----------|
| id | SERIAL | ID чата |
| type | VARCHAR(20) | `'private'` или `'group'` |
| name | VARCHAR(255) | Название (для групп) |
| user1_id | INTEGER | Участник 1 (private) |
| user2_id | INTEGER | Участник 2 (private) |
| created_by | INTEGER | Создатель (group) |
| owner_id | INTEGER | Владелец (group) |

---

#### `messages` — Сообщения
| Колонка | Тип | Описание |
|---------|-----|----------|
| message_id | SERIAL | ID сообщения |
| chat_id | INTEGER | ID чата |
| sender_id | INTEGER | Автор |
| text | TEXT | Текст сообщения |
| file_path | VARCHAR(500) | Путь к файлу |
| file_type | VARCHAR(50) | Тип файла |
| created_at | TIMESTAMP | Дата отправки |
| edited_at | TIMESTAMP | Дата редактирования |
| is_deleted | BOOLEAN | Удалено ли |

---

#### `message_reports` — Жалобы
| Колонка | Тип | Описание |
|---------|-----|----------|
| report_id | SERIAL | ID жалобы |
| message_id | INTEGER | На какое сообщение |
| reporter_id | INTEGER | Кто пожаловался |
| reason | TEXT | Причина жалобы |
| status | VARCHAR(20) | `'pending'`, `'dismissed'`, `'actioned'` |
| reviewed_by | INTEGER | Кто рассмотрел (админ) |
| reviewed_at | TIMESTAMP | Дата рассмотрения |

## Инструкция для админа

### Как получить права администратора

1. Зарегистрируйте обычного пользователя через `/register`
2. Войдите под этим пользователем
3. Вручную измените флаг в базе данных:
   ```sql
   UPDATE maxxx.users SET is_admin = TRUE WHERE user_id = <ваш_id>;
   ```
4. Перезайдите в систему (выйдите и войдите снова)

### Как пользоваться админ-панелью

1. Откройте `http://127.0.0.1:8000/admin`
2. Вы увидите три вкладки:

#### Вкладка "Жалобы" 📋
- Отображаются все ожидающие жалобы (`status='pending'`)
- Кнопка "Рассмотреть" открывает модальное окно
- Варианты действий:
  - **Отклонить жалобу**: жалоба помечается как `dismissed`
  - **Забанить автора**: открывается окно ввода причины бана

#### Вкладка "Активные баны" 🚫
- Список всех забаненных пользователей с причинами
- Кнопка "Разбанить" для каждого пользователя
- Разбан заносится в историю

#### Вкладка "История банов" 📜
- Полная история всех банов и разбанов
- Фильтр по ID пользователя
- Отображает: кто, кого, когда, причина

### Важные правила

✅ **Можно:**
- Банить обычных пользователей с указанием причины
- Разбанивать пользователей
- Просматривать историю банов
- Обрабатывать жалобы пользователей

❌ **Нельзя:**
- Банить себя
- Банить других администраторов
- Просматривать админ-панель без флага `is_admin`

## WebSocket подключение

Для работы чата в реальном времени используется WebSocket.

**URL подключения:** `ws://127.0.0.1:8000/ws`

**Параметры:**
- `token`: JWT-токен пользователя

**Пример подключения (JavaScript):**
```javascript
const ws = new WebSocket(`ws://127.0.0.1:8000/ws?token=${accessToken}`);
```

## Лимиты запросов (Rate Limiting)

| Эндпоинт | Лимит |
|----------|-------|
| `/api/register` | 5 запросов в минуту |
| `/api/login` | 10 запросов в минуту |

При превышении возвращается статус `429 Too Many Requests`.

## Безопасность

### Security Headers
Все ответы включают заголовки безопасности:
- `Strict-Transport-Security` (HSTS)
- `X-Frame-Options: DENY` (защита от clickjacking)
- `X-Content-Type-Options: nosniff`
- `X-XSS-Protection: 1; mode=block`
- `Content-Security-Policy`

### Хеширование паролей
Пароли хешируются алгоритмом bcrypt с солью.

### Очистка памяти
Пароли в открытом виде удаляются из памяти сразу после использования.
